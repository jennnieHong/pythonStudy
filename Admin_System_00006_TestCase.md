# TestCase: 사용자 보안정책 (admin_system_00006)

가맹점/본부 백오피스 내 시스템 관리자 권한용 사용자 보안정책 설정 화면(`admin_system_00006`)의 E2E 테스트 케이스 정의서입니다.

---

## 1. 테스트 기본 정보
* **화면 ID**: `admin_system_00006`
* **메뉴 경로**: 시스템관리 > 영업정보시스템 > 사용자 보안정책 (어드민 권한 전용)
* **테스트 계정**: `admin2` / `0000` (시스템 관리자 계정)
* **관련 테이블**: `hmsfns.SECURETB` (사용자 로그인 차단, 비밀번호 주기 및 난이도 설정 테이블)

---

## 2. 테스트 시나리오 및 결과

| ID | 테스트 항목 | 입력 값 / 조건 | 예상 결과 | 판정 |
|:---|:---|:---|:---|:---|
| 1-1 | **보안정책 로딩 검증** | `admin2` 계정으로 로그인 후 `admin_system_00006` 화면 진입 | `SECURETB` 테이블의 단일 설정 행(Row) 데이터를 조회하여 각 입력 필드(잠김 실패횟수, 잠김해제 대기시간 등)에 정상 매핑됨 | **PASS** |

## 3. 데이터베이스 정밀 분석
* **관련 DDL 및 트리거**:
  * 테이블 `hmsfns.SECURETB`는 시스템 내 로그인 보안 설정 정보를 저장하는 단일 설정 보관 테이블입니다.
  * 분석 결과, 해당 테이블에는 **데이터베이스 트리거(Trigger) 및 내장 프로시저(Stored Procedure)가 정의되어 있지 않습니다.**
  * 따라서 변경 시 다른 테이블로의 연쇄 데이터 전파(Depth 2 ~ Depth 3)는 발생하지 않습니다.
* **형변환 에러 검토**:
  * 입력 항목 중 숫자 항목(예: 실패 횟수, 대기 시간 등)은 문자열이 아닌 숫자 타입으로 DB에 바인딩되며, Java 서비스 코드 및 MyBatis XML(`Admin_System_00006_Sql.xml`)에 적절한 파라미터 맵핑이 구성되어 있어 빈 문자열 전송에 따른 형변환 오류 위험성이 존재하지 않음을 검증했습니다.
