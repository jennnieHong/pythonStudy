## 2. 테스트 시나리오 목록

### 2.1 [ST-DB-001] 로그인 성공 후 매장 대시보드 진입
* **테스트 목적**: 매장 권한 계정으로 로그인 시 자동으로 매장용 대시보드가 로드되고 메뉴 권한 필터가 작동하는지 확인.
* **테스트 단계**:
  1. `http://localhost:8080/backoffice/view/login`으로 접속.
  2. ID에 `H1240961C`, PW에 `0000` 입력 후 로그인 버튼 클릭.
* **기대 결과**:
  * 로그인 성공 후 `/backoffice/view/main`으로 리다이렉트됨.
  * 본문 영역에 매장용 대시보드 뷰(`st/dashboard/dashboard.jsp`)가 동적 로드됨.

### 2.2 [ST-DB-002] 매장 금일 매출 요약 조회
* **테스트 목적**: 본사 전체가 아닌, 로그인한 매장(`NC0007`) 단일 매장의 금일 매출 정보만 집계되어 표시되는지 검증.
* **테스트 단계**:
  1. 대시보드 메인의 당일 매출 카드를 확인.
* **기대 결과**:
  * 타 매장의 매출은 합산되지 않고, `NC0007` 매장의 당일 정산액만 정확히 노출됨.

### 2.3 [ST-DB-003] 매장 인기 상품 순위 조회
* **테스트 목적**: 로그인 매장 기준의 인기 판매 상품 데이터가 차트 및 리스트에 정확히 표현되는지 검증.
* **테스트 단계**:
  1. 상품 판매 순위 그래프 및 데이터 목록 확인.
* **기대 결과**:
  * 해당 매장에서 판매된 상품 코드와 수량 기준의 순위가 리스트에 정상 바인딩됨.

### 2.4 [ST-DB-004] 공지사항 목록 연동
* **테스트 목적**: 본사에서 등록한 공지사항이 매장 대시보드에서도 정상적으로 연동 노출되는지 확인.
* **테스트 단계**:
  1. 대시보드 공지사항 영역 확인.
* **기대 결과**:
  * 본사 공지사항 목록이 정상 조회되며, 매장 사용자가 확인할 수 있어야 함.
