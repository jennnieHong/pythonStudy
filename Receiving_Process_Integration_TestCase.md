# Receiving Process Integration Test Case (입고 E2E 통합 테스트케이스)

본 문서는 HMS 영업정보시스템의 **매입입고 등록/확정 ➡️ 실시간 재고 연동 ➡️ 입고현황 집계**에 이르는 입고 비즈니스 라이프사이클의 정합성을 검증하기 위한 E2E 통합 테스트케이스 명세서입니다.

---

## 1. 테스트 기본 정보

* **테스트 목적**: EDB PostgreSQL 마이그레이션 이후, 입고 전표 상태 변경에 따른 데이터베이스 정합성과 화면 조회 및 집계 연동이 정상적으로 작동하는지 통합 검증한다.
* **테스트 계정 및 소속 매장**:
  * 매장 계정: `fnbcafe` (매장코드: `NC0007` - 고양 모터스튜디오)
  * 본사 계정: `hq_admin` (본사 권한 계정)
* **테스트 대상 상품**:
  * `T0000001` (Food그레놀라 치즈, 입수량: 1, 공급가: 35,000원, 거래처: `000001`)

## 3. 주요 확인 쿼리 (SQL Verification)

E2E 통합 테스트 수행 시 DB 정합성을 수동으로 검증하기 위한 쿼리 템플릿입니다.

### 3.1. 전표 상태값 검증
```sql
SELECT SLIP_NO, SLIP_DATE, MS_NO, VD_NO, SLIP_FG, PROC_FG 
  FROM HMSFNS.OBSLPHTB 
 WHERE SLIP_DATE = TO_CHAR(SYSDATE, 'YYYYMMDD') 
   AND MS_NO = 'NC0007';
```

### 3.2. 실시간 현재고 반영 검증
```sql
SELECT MS_NO, GOODS_CD, CUR_QTY 
  FROM HMSFNS.IMCRIOTB 
 WHERE MS_NO = 'NC0007' 
   AND GOODS_CD = 'T0000001';
```

### 3.3. 수불로그 대기 데이터 검증
```sql
SELECT PROC_DATE, MS_NO, GOODS_CD, PROC_FG, TRLG_QTY 
  FROM HMSFNS.IMTRLGTB 
 WHERE PROC_DATE = TO_CHAR(SYSDATE, 'YYYYMMDD')
   AND MS_NO = 'NC0007';
```
