# HMS 데이터베이스 명세서 사용자 메모 관리 가이드

이 문서는 `hms_db_dictionary.html` 화면에서 작성하는 **사용자 커스텀 메모(테이블 메모 및 컬럼 메모)**의 작동 원리와 안전하게 영구 저장(백업 및 반영)하는 방법에 대해 설명합니다.

---

## 1. 메모 저장 작동 원리

웹 브라우저 상에서 작성한 메모는 **두 가지 방식**으로 관리됩니다:

1. **브라우저 로컬 저장소 (localStorage)**:
   - 화면에서 메모를 [편집] -> [저장]하면 즉시 웹 브라우저 자체의 `localStorage` 영역에 보관됩니다.
   - 브라우저를 새로고침하거나 창을 닫아도 메모는 유지됩니다.
   - **⚠️ 주의**: 브라우저의 인터넷 사용 기록(쿠키, 사이트 데이터, 캐시 등)을 삭제하면 `localStorage`에 저장된 모든 메모가 유실됩니다.

2. **HTML 파일 내장 데이터 (Embedded JSON)**:
   - 파이썬 스크립트(`generate_table_dictionary.py`)가 실행될 때 `table_memos.json` 파일을 읽어와 HTML 내부 코드에 메모를 물리적으로 융합(Embed)시킵니다.
   - 이렇게 내장된 메모는 브라우저 캐시를 날리거나, 다른 사람에게 HTML 파일을 전송하여 열더라도 **영구적으로 보존**됩니다.

---

## 2. 메모 백업 및 영구 반영 순서 (유실 방지 프로세스)

작성한 메모를 안전하게 보관하고 다른 환경에서도 볼 수 있게 하려면 아래 프로세스를 권장합니다:

### 1단계: 브라우저 화면에서 메모 작성
- `hms_db_dictionary.html` 화면을 열어 테이블 메모와 컬럼 메모를 입력 및 저장합니다.

### 2단계: 사용자 메모 백업 (JSON 다운로드)
- 화면 우측 상단의 **`📝 사용자 메모 백업 (JSON)`** 버튼을 클릭합니다.
- `table_memos.json` 파일이 다운로드됩니다.

### 3단계: 백업 파일을 지정된 경로에 이동
- 다운로드한 `table_memos.json` 파일을 아래의 경로로 이동(덮어쓰기)합니다.
  - **저장 경로**: `D:\hmTest\backoffice\QaReport\table_memos.json`

### 4단계: 파이썬 스크립트 실행 (HTML 재빌드)
- 프로젝트 작업 경로에서 명세서 생성 스크립트를 실행합니다.
  ```powershell
  python generate_table_dictionary.py
  ```
- 스크립트가 실행되면서 `table_memos.json` 파일을 읽어와 한글 화면명/배치명 정보 등과 함께 `hms_db_dictionary.html` 파일에 영구 매핑하여 새로 빌드합니다.

---

## 3. 요약 및 주의사항

- **메모의 우선순위**: 
  - 화면에 보여줄 때 **브라우저 로컬 저장소(`localStorage`)에 저장된 값**을 최우선으로 보여주며, 캐시가 없는 경우에만 **HTML 내장 데이터**를 읽어옵니다.
- **백업 파일 유지 필수**:
  - `table_memos.json` 파일을 삭제하고 파이썬 빌드를 다시 실행하면, 새로 빌드된 HTML 파일 내부에는 메모 내용이 모두 누락됩니다. 따라서 백업 파일은 지우지 말고 지정된 위치에 계속 덮어쓰기 형태로 유지해야 합니다.
