# POS 정산 데이터 생성 및 마감 수집 흐름 명세 (POS Settlement Lifecycle)

본 문서는 POS 정산내역 화면에서 조회되는 데이터(`SAREGITB`)가 **원천적으로 어떻게 생성되어 적재(누적)되는지**, 그리고 **어떤 화면과 배치 작업을 거쳐 최종 매장 마감으로 귀결되는지** 그 라이프사이클을 기술합니다.

---

## 1. POS 정산 데이터 라이프사이클 요약

POS 정산 데이터는 매장에서 발생한 개별 결제 로그가 백오프셋에 쌓이는 것과 별개로, **매장 현장의 POS 단말기에서 수행되는 "POS 마감 정산" 작업을 기점으로 생성**됩니다. 이후 백오프리스 시스템인터페이스 화면 및 야간 배치를 거쳐 본사 데이터베이스에 최종 동기화됩니다.

```
[1단계: 매장 현장]                    [2단계: 데이터 전송]                [3단계: 본사 백오프리스]
POS 단말기 결제 (STRNHDTB)          Tibero 매장 DB 동기화              시스템인터페이스 (hq_sysif_00001)
     │                                     │                                    │
     ▼                                     ▼                                    ▼
영업종료 후 "POS 정산마감"          배치 솔루션 (hyundai-batch)         수집 데이터 정합성 대조 (수량/금액)
(실물 시재 세어 입력)               오버나이트 배치 처리                         │
     │                                     │                                    ▼
     ▼                                     ▼                               "현장마감(Store Close)" 실행
SAREGITB (정산데이터 생성)  =======> EDB 본사 DB로 복제 (SAREGITB)  ===> IFSLRETB 마감완료 처리 (Y)
```

---

## 2. 세부 단계별 데이터 누적 흐름

### [1단계] 매장 현장 - POS 단말기 영업 및 정산 마감
1. **결제 및 원천 로그 적재**:
   - 매장 영업 시간 동안 고객이 결제할 때마다 POS 단말기는 개별 거래 정보(영수증 헤더 및 상세 내용)를 매장 로컬 DB의 `STRNHDTB` (거래 헤더) 및 `STRNDTTB` (상품 거래 상세)를 비롯한 결제 수단별 세부 테이블(`STRNCDTB` 카드상세, `STRNCPTB` 쿠폰상세 등)에 실시간으로 저장합니다.
2. **영업 종료 후 POS 마감 실행 (★ SAREGITB 생성 시점)**:
   - 매장 근무자가 영업 종료 후 POS 단말기에서 **[POS 정산]** 또는 **[일마감]** 메뉴를 실행합니다.
   - 이때 포스기는 오늘 해당 POS에서 발생한 총매출액, 카드사별 승인액, 쿠폰 할인액 등을 자체 계산하여 화면에 보여줍니다.
   - 근무자는 돈통(Drawer)에서 현금을 꺼내 권종별 수량(5만원권 N매, 1만원권 N매 등) 및 수표 수량을 직접 세어 단말기에 입력합니다.
   - 입력이 완료되고 **[마감 확정]** 버튼을 누르는 순간, POS 단말기는 입력된 현금 시재와 매출 데이터를 포맷팅하여 **`hmsfns.SAREGITB` 테이블에 `REGI_TYPE = '0'` (일반 마감) 유형으로 1건의 마감 레코드를 Insert**합니다.

---

### [2단계] 데이터 전송 - 매장 DB에서 본사 DB로 동기화 (Batch)
* **배치 가동 (`hyundai-batch`)**:
  - 개별 매장의 로컬 DB(보통 Tibero 또는 로컬 DBMS)에 쌓인 하루 동안의 거래 로그(`STRNHDTB`)와 정산 로그(`SAREGITB`)는 야간 배치 프로그램(`hyundai-batch`) 또는 데이터 동기화 모듈을 통해 본사의 중앙 EDB PostgreSQL DB로 전송(Replication/Insert)되어 누적됩니다.
  - 이 과정을 거치면 본사의 `hmsfns.SAREGITB` 테이블에 당일 매출 정산 데이터가 최종 취합됩니다.

---

### [3단계] 본사/매장 백오프리스 - 정산 수집 및 마감 검증
본사 관리자나 점포 관리자는 전송된 정산 데이터를 검증하고 매장의 영업을 최종 승인(마감)하는 작업을 백오프리스 화면에서 수행합니다.

* **관련 백오프리스 화면**: **`hq_sysif_00001` (시스템인터페이스 > POS 정산 수집/마감)**

#### 검증 및 마감 처리 단계:
1. **마감 내역 조회**:
   - 관리자가 해당 화면에서 특정 일자와 매장을 선택해 조회를 수행합니다.
2. **영수 건수 및 금액 정합성 대조 (`checkBillCnt` 쿼리 가동)**:
   - 시스템은 백오프리스에 수집된 개별 거래 내역(`STRNHDTB` 합계)과 POS 단말기에서 전송한 정산 마감 내역(`SAREGITB` 합계)을 대조합니다.
   - **대조 쿼리 (`checkBillCnt`)**:
     ```sql
     -- 1) 거래 원장(STRNHDTB) 기준 총건수 및 총매출액 계산
     SELECT MS_NO, POS_NO, COUNT(*) BILL_CNT, SUM(SALE_AMT) SALE_AMT 
       FROM hmsfns.STRNHDTB GROUP BY MS_NO, POS_NO
     -- 2) 정산 마감(SAREGITB) 기준 총건수 및 총매출액 대조
     SELECT MS_NO, POS_NO, SALE_AMT, TO_NUMBER(BILL_NO) BILL_CNT 
       FROM hmsfns.SAREGITB
     -- 3) 두 데이터의 금액이나 건수가 다르면 미매칭(오류) 건으로 검출
     WHERE (A.SALE_AMT <> B.SA_SALE_AMT) OR (A.BILL_CNT <> B.SA_BILL_CNT)
     ```
3. **로그인 건수 체크 (`checkLoginCnt` 쿼리 가동)**:
   - 오픈한 POS 단말기 대비 정산마감(`SAREGITB`) 레코드가 누락 없이 모두 수집되었는지 대조합니다.
4. 마감 승인 처리 및 새벽 배치 연동:
   - 정합성이 일치하여 관리자가 **[현장마감]**을 실행하면 `Hq_Sysif_00001_Mapper.closeRegi`가 가동되어 `hmsfns.IFSLRETB` 테이블의 `MS_CLOSE_YN`이 **`'Y'` (마감완료)** 상태로 저장됩니다.
   - **새벽 배치 연동 (Targeting)**: 매일 새벽 구동되는 정기 정산 배치(`hyundai-batch`)는 `IFSLRETB` 테이블에서 `MS_CLOSE_YN = 'Y'` 상태인 레코드만 조회하여 후속 집계 및 ERP 전송, 재고 감액을 처리합니다. **[현장마감]을 누르지 않아 `'N'` 상태인 점포는 새벽 배치의 대상에서 제외**되어 정산이 보류됩니다.
   - 만약 문제가 발생하여 **[재마감]**을 실행하면 상태가 임시 해제되어 수정 후 다시 마감하여 다음 배치 주기의 대상이 되도록 조정할 수 있습니다.
   - **재마감(Re-closing) 제한 조건 및 차단 정책 (★ 배치 가동 후 재마감 불가 로직)**:
      - 재마감(`ReSaleInterFace` API) 처리 시, 시스템은 Tibero DB의 마감 상태 테이블을 대조하여 이미 마감 처리가 완료된 경우 재마감을 차단합니다.
      - **일 마감 여부 대조 (`TSM_SALS_DAYCLS_MST` 테이블)**: 
        - `CLS_YN = 'Y'` (영업일의 최종 매출 마감이 완료된 상태)인 경우, API는 `cls`를 리턴하며 화면에 **"업장 마감이 되어 재마감 처리가 불가합니다."** 경고 팝업을 띄우고 차단합니다.
      - **배치 마감 여부 대조 (`TSM_SHOP_DAYCLS_MST` 테이블)**:
        - `PLCMNT_CLS_YN = 'Y'` (새벽 배치가 구동되어 매장 배치 마감이 완료된 상태)인 경우, API는 `plcmnt`를 리턴하며 화면에 **"업무담당자를 통해 변경 후 재마감처리가 가능합니다."** 경고 팝업을 띄우고 차단합니다.
      - **대처 방안**: 새벽 배치 등이 돌아서 위 플래그들이 `'Y'`로 업데이트된 이후에는 화면을 통한 직접 재마감이 불가능합니다. 이를 변경해야 할 경우 본사 업무담당자를 통해 DB에서 직접 마감 플래그를 해제(미마감 상태로 변경)한 후 재마감을 진행해야 합니다.

## 3. 핵심 요약 요약표

| 단계 | 수행 위치 / 화면 | 관련 테이블 | 발생 작업 및 데이터 흐름 |
| :--- | :--- | :--- | :--- |
| **1. 거래 발생** | 매장 POS 단말기 영업 | `STRNHDTB` | 고객 결제 시 실시간 거래 데이터 적재 |
| **2. 단말기 정산** | 매장 POS 단말기 [POS 마감] | `SAREGITB` | 근무자가 **실물 현금 시재 수량 입력** 시 정산 데이터 적재 |
| **3. 본사 수집** | 스케줄러 배치 (`hyundai-batch`) | `SAREGITB` (EDB) | 매장 로컬 DB -> 본사 EDB 중앙 DB로 배치 이관 및 취합 |
| **4. 대조 및 승인** | **`hq_sysif_00001` (POS정산수집/마감)** | `IFSLRETB` | 원장과 정산 데이터 자동 비교 후 **[현장마감]** 클릭 시 마감 완료(`MS_CLOSE_YN = 'Y'`) 처리 |
