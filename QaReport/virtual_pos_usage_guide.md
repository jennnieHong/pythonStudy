# HMS 가상 포스 시뮬레이터(Virtual POS) 사용 가이드

본 가이드는 포스 단말기 장비 없이, 가상 데이터를 통해 **[포스 매출 발생 -> 백오피스 수집/대조 -> 매장 현장마감 -> 야간 배치 정산 -> 재고 감액 및 수불로그 적재]** 흐름을 통합 테스트하기 위한 `virtual_pos_simulator.py` 스크립트의 사용법을 정리합니다.

---

## 1. 개요 및 설계 원리

백오피스 화면 중 **`hq_sysif_00001` (POS 정산 수집/마감)** 메뉴는 포스기가 하루 마감 시 전송한 **정산마감 데이터(`SAREGITB`)**와 당일 발생한 **거래 원장 데이터(`STRNHDTB`)**를 가져와 수량 및 금액의 정합성 대조(Audit)를 수행합니다. 
두 데이터의 총액과 건수가 1원/1건이라도 일치하지 않으면 미매칭(불일치) 오류로 인해 마감이 불가능합니다.

가상 포스 시뮬레이터는 지정된 매출액에 대해 **거래 데이터와 마감 데이터를 금액상 완벽히 일치시켜 적재**하여, 관리자가 백오피스에서 즉시 **[현장마감]**을 실행하고 새벽 배치를 가동할 수 있는 상태로 만들어 줍니다.

---

## 2. 사전 준비 사항

스크립트 구동을 위해 PostgreSQL 연동 라이브러리(`psycopg2` 또는 `psycopg2-binary`)가 설치되어 있어야 합니다.
```powershell
pip install psycopg2-binary
```

## 3. 시뮬레이터 사용법 (명령어 가이드)

`virtual_pos_simulator.py` 스크립트는 실행 시 다양한 파라미터를 인수로 받아 맞춤형 매출 데이터를 생성할 수 있습니다.

### 3.1 기본 실행 (Default)
오늘 날짜, NC0007(CAFE) 매장, 01번 포스기 기준으로 총 매출 50,000원(현금 10,000원, 카드 40,000원, 준비금 100,000원) 데이터를 적재합니다.
```powershell
python virtual_pos_simulator.py
```

### 3.2 날짜 및 매장 커스텀 실행
특정 날짜(`--date`), 특정 매장코드(`--ms_no`)를 지정하여 실행합니다.
```powershell
python virtual_pos_simulator.py --date 20260612 --ms_no NC0007
```

### 3.3 매출액 및 현금/카드 비중 세부 지정
총매출액(`--sale_amt`)과 현금 비중(`--cash_amt`), 포스 준비금(`--prep_amt`)을 지정하여 테스트 데이터를 생성합니다. (나머지 잔액은 자동으로 신용카드 매출로 입력됩니다.)
```powershell
# 총 매출 150,000원 (현금 30,000원, 카드 120,000원, 준비금 200,000원)
python virtual_pos_simulator.py --date 20260612 --ms_no NC0007 --sale_amt 150000 --cash_amt 30000 --prep_amt 200000
```

### 3.4 명령어 옵션 목록

| 옵션명 | 설명 | 기본값 | 비고 |
| :--- | :--- | :--- | :--- |
| `--date` | 영업일자 (YYYYMMDD) | 오늘 날짜 | 예: `20260612` |
| `--ms_no` | 매장 번호 (매장 마스터 코드) | `NC0007` | MMEMBSTB 테이블에 존재해야 함 |
| `--pos_no` | POS 번호 | `01` | 자리수 맞춰 자동 2자리 패딩 |
| `--sale_amt` | 총매출액 (할인 전 금액) | `50000.0` | 총 판매 금액 |
| `--cash_amt` | 현금 결제액 | `10000.0` | 실물 금고 시재 영향 |
| `--prep_amt` | 오픈 준비금 (거스름돈) | `100000.0` | 정산 시재 기초값 |
| `--bill_no` | 최종 영수증 일련번호 (거래 건수) | `0001` | 자리수 맞춰 자동 4자리 패딩 |

## 4. 연동 통합 테스트 시나리오 수행 절차

가상 데이터를 활용해 흐름을 완벽히 E2E 검증하려면 아래 단계에 따라 테스트를 완료합니다.

### 1단계: 가상 포스 실행
테스트를 원하는 매장과 일자 정보를 담아 스크립트를 실행합니다.
```powershell
python virtual_pos_simulator.py --date 20260612 --ms_no NC0007 --sale_amt 80000 --cash_amt 20000
```
*성공 시 기존 시뮬레이션용 중복 키 데이터는 자동으로 Delete 처리된 후 Insert 됩니다.*

### 2단계: 백오피스 정산 수집 화면 확인
1. 본사 관리자 계정(`shopadmin` 또는 `fnbadmin`)으로 백오피스 로그인
2. **[시스템인터페이스] > [POS 정산 수집/마감]** (`hq_sysif_00001`) 화면 진입
3. 일자: `2026-06-12`, 매장: `CAFE` 조회
4. 매출 총액(80,000원), 현금/카드/부가세 및 영수건수 정합성이 완벽히 대조(Match) 통과 상태인지 확인합니다.

### 3단계: 매장 현장마감(Store Close) 클릭
1. 화면에서 해당 매장의 체크박스를 선택한 후 우측 상단의 **[현장마감]** 버튼을 실행합니다.
2. EDB Database에서 아래 쿼리를 구동하여 `MS_CLOSE_YN` 필드가 `'Y'`로 업데이트되었는지 조회합니다.
   ```sql
   SELECT ms_no, sale_date, ms_close_yn, last_dtime, last_id 
     FROM hmsfns.IFSLRETB 
    WHERE sale_date = '20260612' AND ms_no = 'NC0007';
   ```

### 4단계: 새벽 야간 정산 배치(`hyundai-batch`) 수동 구동
- 로컬 또는 개발 서버 환경에서 매장 매출 확정 및 재고를 차감하는 새벽 배치를 트리거시킵니다.
- 배치는 `IFSLRETB.MS_CLOSE_YN = 'Y'`인 NC0007 매장의 판매 데이터를 분석하여 후속 재고 반영 작업을 수행합니다.

### 5단계: 현재고 및 수불 로그(MGOODSTB, MMSLOGTB) 확인
1. **현재고 감액 대조**: 판매된 제품의 단품/세트 구조에 따라 가맹점 상품 마스터 재고가 정상 차감되었는지 검증합니다.
   ```sql
   SELECT ms_no, goods_cd, cur_qty 
     FROM hmsfns.MGOODSTB 
    WHERE ms_no = 'NC0007' AND goods_cd = '테스트상품코드';
   ```
2. **수불로그 원장 기록 검증**: 판매 수량 및 매출 유형(`POS 매출`)이 로그 데이터로 정상 누적되었는지 확인합니다.
   ```sql
   SELECT ms_no, sale_date, log_fg, goods_cd, qty, bill_no 
     FROM hmsfns.MMSLOGTB 
    WHERE sale_date = '20260612' AND ms_no = 'NC0007';
   ```
