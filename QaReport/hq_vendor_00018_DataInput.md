## 5. 원가율(%) 및 순매출액(NET_SALE_AMT) 산정 공식 상세 가이드

본 화면(`hq_vendor_00018`)에서 사용하는 집계 산식 및 데이터 정합성 검증 기준입니다.

### 5.1 원가율(%) 계산 공식 및 구성
화면 그리드 및 보고서에 표시되는 원가율의 근본 공식은 다음과 같습니다.

*   **원가율(%) = (당월 사용 원가 (MONTH_USE_COST) / 순매출액 (SALE_AMT)) * 100**

MyBatis XML 내부에서는 다음과 같은 세부 연산을 통해 각 항목을 구성합니다.

#### ① 분자: 당월 사용 원가 (`MONTH_USE_COST`)
수불 마스터 데이터를 가공하여 실제 소진된 원자재/상품의 원가 총합을 산출합니다.
*   **당월 사용 원가 = (전월 이월 재고액 + 당월 총 매입액 + 매장간 이동 순증가액) - 당월 기말 재고액**
*   **MyBatis 매핑 쿼리**:
    ```sql
    ( ( NVL(SUM(DD.START_COST), 0) + NVL(SUM(DD.PURCH_COST), 0) + (NVL(SUM(DD.MOVE_IN_COST), 0) - NVL(SUM(DD.MOVE_OUT_COST), 0)) )
      - NVL(SUM(DD.END_COST), 0) ) AS MONTH_USE_COST
    ```

#### ② 분모: 순매출액 (`SALE_AMT`)
매출액을 집계할 때, 영수증의 매출 정보(`NET_SALE_AMT`)를 불러와 가산/차감 처리합니다.
*   **MyBatis 매핑 쿼리**:
    ```sql
    DT.NET_SALE_AMT AS SALE_AMT
    ```

#### ③ 최종 연산 및 나눗셈 오류(Zero-Division) 예외 처리
*   **MyBatis 최종 연산식**:
    ```sql
    ROUND(NVL(SUM(RG.MONTH_USE_COST) / DECODE(SUM(RG.SALE_AMT), 0, NULL, SUM(RG.SALE_AMT)), 0) * 100, 2) AS UCOST_RATE
    ```
    *   **Zero-Division 방어**: 순매출액(`SALE_AMT`) 합계가 `0`인 매장은 나누기 오류를 방지하기 위해 `DECODE`를 사용해 분모를 `NULL`로 치환하고, `NVL`을 통해 최종 원가율이 `0`이 되도록 예외 처리합니다.
    *   **반올림**: 최종 백분율(`* 100`) 결과값에 대해 소수점 둘째 자리까지 반올림(`ROUND(..., 2)`) 처리합니다.

### 5.2 순매출액 (`NET_SALE_AMT`)의 상세 산정 방식

#### ① POS 결제 단계에서의 산정 (공급가액 기준)
매출 발생 시 `STRNDTTB` 테이블에 기록되는 순매출액(`NET_SALE_AMT`)은 부가세와 할인금액이 제외된 순수 공급가액을 의미합니다.
*   **순매출액 (NET_SALE_AMT) = 총 매출액 (수량 * 단가) - 에누리/할인금액 - 부가세액 (VAT)**
*   *예시*: 소비자가 11,000원짜리 음료를 구매하고 1,000원 쿠폰 할인을 받아 최종 10,000원을 결제한 경우:
    *   **부가세액 (`VAT_AMT`)**: 결제금액의 10%인 **909원**
    *   **순매출액 (`NET_SALE_AMT`)**: 부가세를 차감한 **9,091원** (이 금액이 매출 집계의 기초값이 됨)

#### ② 화면 조회 집계 단계에서의 가공 (반품 차감 및 중복 조인 배제)
MyBatis Mapper는 조회 시 정상 매출과 반품 취소분을 반영하여 순매출 총계를 냅니다.
*   **집계 처리**:
    ```sql
    SUM(RG.SALE_AMT * DECODE(RG.SALE_FG, 0, 1, -1) * DECODE(RNUM, 1, 1, 0)) AS SALE_AMT
    ```
    *   **반품 거래 차감 (`DECODE(SALE_FG, 0, 1, -1)`)**: 매출구분(`SALE_FG`)이 `0` (정상 판매)이면 매출액을 더하고, `1` (반품/환불)이면 해당 금액만큼 마이너스(-)하여 매출에서 실시간 차감합니다.
    *   **중복 조인 방지 (`DECODE(RNUM, 1, 1, 0)`)**: 영수증 상세 내역에서 발생할 수 있는 데이터 중복 팽창을 배제하기 위해, 영수증 번호별 고유 일련번호(`ROW_NUMBER`)가 `1`인 단일 로우의 값만 매출액에 반영하고 나머지는 `0` 처리합니다.
