## 1단계: 문자열 공백 제거 및 대문자 통일 (String Trimming)

### 💡 무슨 뜻인가요?
사용자가 엑셀에 데이터를 입력할 때 앞뒤에 실수로 넣은 공백(`" nc0002 "`)을 잘라내어 지우고, 소문자로 입력된 것을 모두 대문자(`"NC0002"`)로 일관성 있게 변경하는 단계입니다. 이렇게 해야 데이터 검색이나 DB 매핑 시 매칭 오류가 나지 않습니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: `[" nc0002 ", "NC0007", "nc0005  "]`
* **가공 후 (After)**: `["NC0002", "NC0007", "NC0005"]`

### 💻 따라하기 코드 (`step1_clean.py`)
```python
import pandas as pd

# 1. dirty_sales.xlsx 파일을 메모리로 읽어옵니다. (DataFrame 형태로 변환됨)
df = pd.read_excel("dirty_sales.xlsx")

# 2. '매장코드' 열의 데이터 앞뒤 공백을 없애고(.str.strip()) 대문자로 바꿉니다(.str.upper()).
df["매장코드"] = df["매장코드"].str.strip().str.upper()

# 3. 화면에 가공된 데이터를 출력해 확인해 봅니다.
print(df["매장코드"])
```

### 🔍 코드 한 줄 한 줄 쪼개보기!
* `df["매장코드"]`: 엑셀에서 `'매장코드'`라는 열(Column) 전체를 의미합니다.
* `.str`: 엑셀 셀 안에 있는 데이터들을 하나하나 '문자열(String)' 형태로 다루겠다는 뜻입니다.
* `.strip()`: 문자열 양 끝의 빈 칸(공백)을 도려내는 함수입니다.
* `.upper()`: 영문 소문자를 대문자로 전환해 주는 함수입니다.

## 2단계: 구분자 분할 및 문자열 합치기 (Split & Concat)

### 💡 무슨 뜻인가요?
하나의 셀에 `"SEOUL_SHOP_01"` 처럼 여러 정보가 언더바(`_`)로 묶여 적혀 있을 때, 이를 잘라내어 새로운 열인 `"지역"`과 `"업태"`로 따로따로 저장하고, 필요 시 다시 결합하는 방법입니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: 매장정보 $\rightarrow$ `"SEOUL_SHOP_01"`
* **가공 후 (After)**: 지역 $\rightarrow$ `"SEOUL"`, 업태 $\rightarrow$ `"SHOP"`, 요약 $\rightarrow$ `"SEOUL-SHOP"`

### 💻 따라하기 코드 (`step2_split.py`)
```python
import pandas as pd

df = pd.read_excel("dirty_sales.xlsx")

# 1. 언더바(_) 기호를 기준으로 텍스트를 나눕니다.
# expand=True는 분할된 단어들을 각각 독립된 새 열(Column)로 쪼개어 표 형태로 만든다는 뜻입니다.
split_data = df["매장정보"].str.split("_", expand=True)

# 2. 쪼개진 표의 0번째 칸을 '지역', 1번째 칸을 '업태' 열로 대입합니다.
df["지역"] = split_data[0]
df["업태"] = split_data[1]

# 3. 두 개의 문자열 열을 대시(-) 기호로 묶어서 '매장요약'이라는 새 열을 만듭니다.
df["매장요약"] = df["지역"] + "-" + df["업태"]

# 4. 제대로 가공되었는지 해당 열들만 뽑아서 출력해 봅니다.
print(df[["매장정보", "지역", "업태", "매장요약"]])
```

### 🔍 코드 한 줄 한 줄 쪼개보기!
* `str.split("_", expand=True)`: `_` 기호를 가위 삼아 자릅니다. `expand=True`가 없으면 결과가 리스트 `['SEOUL', 'SHOP', '01']`로 뭉쳐서 나와서 다루기 곤란해집니다.
* `split_data[0]`: 쪼개진 결과 중 첫 번째 영역(지역명)을 의미하는 인덱스입니다.

## 3단계: 제각각인 날짜 포맷 표준화 (Date Standardization)

### 💡 무슨 뜻인가요?
엑셀에 날짜를 적을 때 누구는 슬래시(`2026/07/07`), 누구는 하이픈(`2026-07-07`), 누구는 점(`2026.07.07`)을 찍어서 마구잡이로 관리되는 날짜 형식을 시스템 적재용 표준 규격인 `YYYY-MM-DD` 문자열 형태로 변환하여 정돈합니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: `"2026/07/07"`, `"20260707"`, `"2026.07.07"`
* **가공 후 (After)**: `"2026-07-07"`, `"2026-07-07"`, `"2026-07-07"`

### 💻 따라하기 코드 (`step3_date.py`)
```python
import pandas as pd

df = pd.read_excel("dirty_sales.xlsx")

# 1. pd.to_datetime 함수를 사용하면 어떤 형식이든 날짜 시간 객체로 통일되게 파싱해 줍니다.
date_objects = pd.to_datetime(df["거래일자"])

# 2. 날짜 객체(.dt)를 우리가 원하는 포맷(연도-월-일) 문자열로 변환해 새 열에 저장합니다.
df["정제일자"] = date_objects.dt.strftime("%Y-%m-%d")

print(df[["거래일자", "정제일자"]])
```

### 🔍 코드 한 줄 한 줄 쪼개보기!
* `pd.to_datetime()`: 텍스트 날짜를 파이썬의 연산 가능한 날짜(Timestamp) 객체로 번역해 주는 강력한 자동 번역기 함수입니다.
* `.dt`: 날짜 객체 전용 기능(날짜 포맷 변경, 월/일 추출 등)을 사용하겠다고 활성화하는 도구입니다.
* `.strftime("%Y-%m-%d")`: 날짜 값을 문자열로 표현하되, `%Y`(4자리 연도), `%m`(2자리 월), `%d`(2자리 일)에 하이픈을 넣어 표현하라는 양식 지정입니다.

## 4단계: 빈 데이터(Null/NaN) 검색 및 대체 (Handling Missing Values)

### 💡 무슨 뜻인가요?
엑셀에서 필수로 기록되어야 하는 금액 칸이나 매장코드 등이 비어있어서 `NaN`(Not a Number, 결측치) 상태가 되면 덧셈 연산 시 에러를 유발합니다. 이 빈칸을 감지하여 안전한 기본값(예: 숫자 `0` 또는 문자 `"N/A"`)으로 채워 넣는 방법입니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: `[..., "3,800,000", NaN, None]`
* **가공 후 (After)**: `[..., "3,800,000", "0", "0"]` (빈칸이 "0"으로 정돈됨)

### 💻 따라하기 코드 (`step4_null.py`)
```python
import pandas as pd

df = pd.read_excel("dirty_sales.xlsx")

# 1. 엑셀 파일 내의 누락(Null) 데이터가 각 열마다 몇 개 있는지 숫자를 세어 봅니다.
print("--- 컬럼별 빈칸 개수 ---")
print(df.isnull().sum())

# 2. '금일매출' 열에 비어있는 칸(NaN)이 있으면 문자열 "0"으로 채워 넣습니다.
df["금일매출"] = df["금일매출"].fillna("0")

print("\n--- 빈칸 채우기 후 매출 데이터 ---")
print(df["금일매출"])
```

### 🔍 코드 한 줄 한 줄 쪼개보기!
* `.isnull()`: 각 셀이 비어있으면 `True`, 값이 들어있으면 `False`로 매핑해 주는 함수입니다.
* `.sum()`: 빈칸들의 개수(`True`)를 열 단위로 전부 덧셈하여 출력해 줍니다.
* `.fillna("0")`: 빈 셀(Null)을 찾아 괄호 안의 매개변수인 `"0"` 값으로 채워 넣는 치환 함수입니다.

## 5단계: 문자 섞인 매출액을 숫자로 변환 (Type Casting)

### 💡 무슨 뜻인가요?
엑셀의 매출액이 콤마 문자(`"5,200,000"`)나 수치가 아닌 특수문자 입력(`"N/A"`)으로 기재되어 있으면 파이썬에서 합계나 평균 연산을 수행할 수 없습니다. 문자열 기호를 지우고, 컴퓨터가 산술 연산을 할 수 있는 정수 타입(`int`)으로 변환합니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: `"5,200,000"` (계산 불가능한 문자열)
* **가공 후 (After)**: `5200000` (계산 가능한 정수 수치)

### 💻 따라하기 코드 (`step5_cast.py`)
```python
import pandas as pd

df = pd.read_excel("dirty_sales.xlsx")

# 1. 먼저 빈칸을 0으로 채워넣기 안전 처리를 합니다.
df["금일매출"] = df["금일매출"].fillna("0")

# 2. 텍스트 형식으로 변환하여 천 단위 콤마(,)를 빈 문자('')로 치환해 삭제합니다.
df["금일매출"] = df["금일매출"].astype(str).str.replace(",", "")

# 3. "N/A" 등 텍스트가 박혀있는 셀도 "0"으로 변경합니다.
df["금일매출"] = df["금일매출"].str.replace("N/A", "0")

# 4. 문자열 타입에서 정수 타입(int)으로 일괄 강제 형변환합니다.
df["금일매출"] = df["금일매출"].astype(int)

# 5. 이제 덧셈이나 평균이 가능해진 결과를 출력하여 연산해 봅니다.
total_sum = df["금일매출"].sum()
print(f"매출 정수 변환 완료! 총 매출 합계: {total_sum:,}원")
```

### 🔍 코드 한 줄 한 줄 쪼개보기!
* `.astype(str)`: 데이터를 일단 강제로 '문자열(String)' 데이터 타입으로 치환합니다. replace 문자 치환 메소드를 사용하기 위한 사전 정지 작업입니다.
* `.str.replace(",", "")`: 쉼표 문자(`,`)를 찾아 지워버립니다.
* `.astype(int)`: 최종 정제된 문자열 숫자(예: `"5200000"`)를 실제 덧셈이 가능한 정수형 타입(`int`)으로 완전히 형변환해 줍니다.

## 6단계: 조건에 따른 등급 파생 열 생성 (np.where)

### 💡 무슨 뜻인가요?
매출 성과를 분류하기 위해, 특정 기준점(예: 매출액 400만 원 초과)을 잡아서 조건이 일치하면 `"VIP"`, 그렇지 않으면 `"일반"`이라는 텍스트 라벨 값을 새로운 열을 만들어 대입하는 방식입니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: 매출 수치 데이터만 존재 (`5,200,000`, `3,800,000`)
* **가공 후 (After)**: `5,200,000` $\rightarrow$ `"VIP"`, `3,800,000` $\rightarrow$ `"일반"`

### 💻 따라하기 코드 (`step6_condition.py`)
```python
import pandas as pd
import numpy as np  # 조건 판단용 라이브러리인 numpy를 호출합니다.

df = pd.read_excel("dirty_sales.xlsx")

# 5단계의 수치 변환 과정을 단축 적용합니다.
df["금일매출"] = df["금일매출"].fillna("0").astype(str).str.replace(",", "").str.replace("N/A", "0").astype(int)

# 조건 설정: 금일매출이 4000000원보다 많은가?
condition = df["금일매출"] > 4000000

# np.where(조건문, 참일때적용할값, 거짓일때적용할값)
df["고객등급"] = np.where(condition, "VIP", "일반")

print(df[["금일매출", "고객등급"]])
```

### 🔍 코드 한 줄 한 줄 쪼개보기!
* `np.where(A, B, C)`: 엑셀의 `IF` 함수(`=IF(매출>400만, "VIP", "일반")`)와 완벽하게 대칭되는 기능을 제공하는 파이썬 고속 연산 도구입니다.

## 7단계: 개인정보 보호 연락처 마스킹 (Regex Masking)

### 💡 무슨 뜻인가요?
백오피스 리포트 생성 시 개인정보 유출을 방지하기 위해, 정규식 매칭을 이용해 전화번호의 중간 4자리(예: `1234`)를 별표(`****`)로 자동 치환하는 개인정보 가공 기술입니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: `"010-1234-5678"`
* **가공 후 (After)**: `"010-****-5678"`

### 💻 따라하기 코드 (`step7_mask.py`)
```python
import pandas as pd
import re  # 정규표현식 제어 모듈 호출

df = pd.read_excel("dirty_sales.xlsx")

# 번호 패턴을 찾아서 치환해주는 마스킹 전용 함수 정의
def apply_mask(phone_number):
    phone_str = str(phone_number).strip()
    
    # 010-숫자4개-숫자4개 패턴을 감지하여 가운데 영역만 별표로 치환합니다.
    # (\d{3}) -> 1번그룹(숫자3개), (\d{4}) -> 2번그룹(숫자4개), (\d{4}) -> 3번그룹(숫자4개)
    pattern = r'(\d{3})-(\d{4})-(\d{4})'
    
    # \1-****-\3 은 1번그룹(010)과 3번그룹(마지막4자리)은 살리고 가운데 2번그룹만 ****로 바꾸라는 정규식 치환문입니다.
    masked = re.sub(pattern, r'\1-****-\3', phone_str)
    return masked

# apply() 메소드를 사용하여 연락처 열의 모든 데이터행에 위 마스킹 함수를 일괄 작동시킵니다.
df["고객연락처"] = df["고객연락처"].apply(apply_mask)

print(df["고객연락처"])
```

## 8단계: 그룹 통계 데이터 집계 (GroupBy)

### 💡 무슨 뜻인가요?
전체 매장 데이터가 있을 때, `"지역별"`로 분류해서 서울 매장들의 전체 총매출 합계가 얼마이고, 총 매장 수가 몇 개가 등록되어 있는지 자동으로 묶어서 요약 보고서 테이블로 산출하는 기능입니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: 개별 거래 내역 리스트 5행
* **가공 후 (After)**: 지역별 요약 집계 표 (서울: 총매출 1040만 원 / 매장수 2개 등)

### 💻 따라하기 코드 (`step8_groupby.py`)
```python
import pandas as pd

df = pd.read_excel("dirty_sales.xlsx")
df["금일매출"] = df["금일매출"].fillna("0").astype(str).str.replace(",", "").str.replace("N/A", "0").astype(int)

# 지역 데이터 쪼개어 생성 (2단계 로직 적용)
df["지역"] = df["매장정보"].str.split("_", expand=True)[0]

# groupby("지역")은 지역이 같은 데이터끼리 묶어서 분리 수납하는 행위입니다.
# agg(...)는 그 수납된 그룹들 안에서 어떤 통계(합계, 개수 등)를 연산할지 명령하는 메소드입니다.
summary_report = df.groupby("지역").agg(
    총매출합계=("금일매출", "sum"),
    등록매장수=("금일매출", "count")
).reset_index() # 인덱스를 깔끔한 컬럼 구조로 원복 정돈

print(summary_report)
```

## 9단계: 중복 데이터 제거 및 고유값 유지 (Drop Duplicates)

### 💡 무슨 뜻인가요?
시스템 연동이나 네트워크 중복 패킷 전송 등으로 인해 엑셀 시트에 동일한 거래 데이터가 중복으로 기재된 경우, 중복 행을 찾아내어 과다 청구/집계를 막기 위해 1행만 보존하고 나머지는 삭제(Drop)해 버리는 기법입니다.

### 📝 데이터의 상태 변화
* **가공 전 (Before)**: `NC0002` 매장의 7월 7일 거래행이 2번 중복 적재되어 있음.
* **가공 후 (After)**: 동일한 중복 거래행이 소거되어 단 1행만 유일값으로 남음.

### 💻 따라하기 코드 (`step9_duplicate.py`)
```python
import pandas as pd

df = pd.read_excel("dirty_sales.xlsx")

# drop_duplicates 메소드는 중복 데이터를 자동 감지해 삭제합니다.
# subset: 중복인지 비교 검사할 기준 열 지정 (매장코드와 거래일자가 완벽히 겹치면 중복 판정)
# keep='first': 중복된 행 중 가장 먼저 나온 첫 번째 행은 남기고 뒤의 것들을 버립니다.
df_cleaned = df.drop_duplicates(subset=["매장코드", "거래일자"], keep="first")

print("--- 원본 행 개수 ---:", len(df))       # 5행
print("--- 중복 제거 후 행 개수 ---:", len(df_cleaned)) # 4행
```

## 10단계: 최종 가공 데이터 엑셀 파일 저장 및 검증 (Export)

### 💡 무슨 뜻인가요?
1단계부터 9단계까지의 모든 데이터 클리닝 처리 로직을 하나의 온전한 가공 파이프라인 스크립트로 완성하여, 정제된 최종본 데이터를 `"final_clean_report.xlsx"` 라는 정형화된 새 엑셀 시트로 깨끗하게 저장하는 마무리에 도달합니다.

### 💻 최종 통합 실행 코드 (`step10_final.py`)
아래 코드를 복사하여 **`step10_final.py`** 파일을 만들고 최종 실행해 보세요!

```python
# step10_final.py
import pandas as pd
import numpy as np
import re

def main():
    print("1. [데이터 로드] 원본 dirty_sales.xlsx 파일을 엽니다.")
    df = pd.read_excel("dirty_sales.xlsx")
    
    print("2. [매장코드 정제] 양끝 공백 제거 및 영문 대문자 통일")
    df["매장코드"] = df["매장코드"].str.strip().str.upper()
    
    print("3. [매장정보 파싱] 지역과 업태 속성 분할 추출")
    split_cols = df["매장정보"].str.split("_", expand=True)
    df["지역"] = split_cols[0]
    df["업태"] = split_cols[1]
    
    print("4. [날짜 포맷 정형화] 일정한 YYYY-MM-DD 형태로 변환")
    df["거래일자"] = pd.to_datetime(df["거래일자"]).dt.strftime("%Y-%m-%d")
    
    print("5. [매출액 수치화] 콤마 삭제 및 결측치 0원 처리 후 정수형 변환")
    df["금일매출"] = df["금일매출"].fillna("0")
    df["금일매출"] = df["금일매출"].astype(str).str.replace(",", "").str.replace("N/A", "0").astype(int)
    
    print("6. [파생 변수 정의] 매출 기준에 따라 VIP/일반 등급 판정 적용")
    df["고객등급"] = np.where(df["금일매출"] > 4000000, "VIP", "일반")
    
    print("7. [개인정보 처리] 연락처 중간 4자리 정규식 마스킹 처리")
    df["고객연락처"] = df["고객연락처"].apply(
        lambda x: re.sub(r'(\d{3})-(\d{4})-(\d{4})', r'\1-****-\3', str(x).strip())
    )
    
    print("8. [중복 중복 제거] 중복 등록된 불량 행 드롭 처리")
    df_clean = df.drop_duplicates(subset=["매장코드", "거래일자"], keep="first")
    
    # 9. 필요없는 예전 '매장정보' 열은 빼고 정리된 최종 컬럼 포맷 지정
    final_columns = ["매장코드", "지역", "업태", "거래일자", "금일매출", "고객등급", "고객연락처"]
    df_final = df_clean[final_columns]
    
    print("9. [엑셀 파일 저장] 최종 정제 완료 보고서 파일 쓰기")
    df_final.to_excel("final_clean_report.xlsx", index=False)
    
    print("\n🎉 모든 데이터 정제 파이프라인 처리가 성공적으로 완료되었습니다!")
    print("생성된 파일명: final_clean_report.xlsx")

if __name__ == "__main__":
    main()
```
