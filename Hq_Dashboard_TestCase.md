# Test Case: Hq_Dashboard 대시보드 (본사)
**작성일**: 2026-07-06  
**작성자**: AI QA Agent (Antigravity)  
**대상 화면**: [HQ] 대시보드 (hq_dashboard)

---

## 1. 테스트 준비 사항

### 1.1 기본 환경 정보
* **최초 접속 URL**: `http://localhost:8080/backoffice/view/login`
* **테스트 로그인 계정**: `shopadmin` (비밀번호: `0000`, 체인번호: `C001`)

### 1.2 사전 조건 및 데이터 상태
* 데이터베이스에 로그인 사용자 `shopadmin` 정보가 등록되어 있어야 함.
* `SDAILYTB` (매출 일집계 테이블)에 당일 매출 데이터가 적재되어 있어야 당일 매출 그래프가 정상 로드됨.
* `MGOODSTB` (상품 마스터) 및 `SDAILYTB`에 상품별 매출 내역이 존재해야 상품 판매 순위 그래프가 정상 로드됨.
* `TNOTICETB` (공지사항 테이블)에 공지사항 데이터가 적재되어 있어야 공지사항 목록이 로드됨.

---

## 2. 테스트 시나리오 목록

### 2.1 [HQ-DB-001] 로그인 성공 후 메인 대시보드 진입
* **테스트 목적**: 본사 권한 계정으로 로그인 시 자동으로 본사 대시보드 페이지가 로드되고 메뉴가 정상 활성화되는지 검증.
* **테스트 단계**:
  1. `http://localhost:8080/backoffice/view/login`으로 접속.
  2. ID에 `shopadmin`, PW에 `0000` 입력 후 로그인 버튼 클릭.
* **기대 결과**:
  * 로그인 성공 후 `/backoffice/view/main`으로 리다이렉트됨.
  * URL 경로가 정상 유지되며, 본문 영역에 본사 대시보드 뷰(`hq/dashboard/dashboard.jsp`)가 로드됨.

### 2.2 [HQ-DB-002] 금일 매출 요약 정보 조회
* **테스트 목적**: 상단 매출 요약 영역에 금일 매출 금액, 주문 건수, 실시간 정산액 등이 정상 수치로 바인딩되는지 확인.
* **테스트 단계**:
  1. 로그인 성공 후 메인 대시보드 페이지 로드 확인.
  2. 금일 매출 현황 카드의 금액 및 수치 확인.
* **기대 결과**:
  * 금일 매출 정보가 `0원` 또는 데이터베이스 실제 당일 정산 합계액으로 정상 표시됨.

### 2.3 [HQ-DB-003] 판매 순위 및 차트 렌더링
* **테스트 목적**: 대시보드 내부의 ApexCharts 기반 판매 추이 그래프 및 상위 판매 상품 목록이 정상 렌더링되는지 확인.
* **테스트 단계**:
  1. 대시보드 본문의 차트 및 그리드 영역 확인.
* **기대 결과**:
  * 매출 추이 차트가 깨지지 않고 정상 표시됨.
  * 상품 판매 순위(1위~5위) 테이블이 정상적으로 바인딩되어 출력됨.

### 2.4 [HQ-DB-004] 공지사항 목록 연동
* **테스트 목적**: 최근 공지사항 목록(최대 5건)이 정상적으로 연동되어 그리드에 노출되는지 확인.
* **테스트 단계**:
  1. 대시보드 우측 또는 하단의 공지사항 목록 영역 확인.
* **기대 결과**:
  * 최근 등록된 공지사항의 제목, 작성일이 정상 표시됨.
